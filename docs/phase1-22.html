Parcourir le web

Le but du premier travail est d'écrire un programme qui parcourt des pages web en traversant à chaque fois le premier hyperlien qu'on y trouve.

En plus de parcourir les pages, l'application devra

  • Sauvegarder localement les pages explorées
  • Collecter les adresses courriels trouvées dans les pages

Gestion des arguments

Arguments en ligne de commandes

Ton application doit prendre 2 arguments en ligne de commande dans l'ordre spécifié ici.

Attention! Il ne s'agit pas d'une application intéractive. Tu dois passer les paramètres en ligne de commandes. Il est fortement recommandé de créer plusieurs configurations d'exécution dans Intellij IDEA pour tester plusieurs combinaisons de paramètres corrects ou incorrects.

1 point Dans le cas où l'utilisateur ne fournit pas le bon nombre d'arguments tu dois lui indiquer clairement comment utiliser l'application.

  • combien d'arguments
  • quel est le rôle de chacun
  • un exemple d'utilisation
Les arguments sont:
  1. 2 points l'url de départ qui est la première page à explorer. Il doit s'agir d'une adresse valide (le format est correct et il y a une page qui y correspond).
  2. 2 points une liste de mots clé permettant d'indiquer les URLs qu'on souhaite ignorer dans l'exploration

Pour chaque argument, tu obtiens un point s'il est bien pris en compte et un point si tu affiches un message significatif quand la valeur n'est pas bonne ou absente.

Traitement

Exploration

3 points Pour chaque étape de l'exploration, on télécharge le html correspondant à l'url actuel

  • on télécharge le html correspondant à l'url actuelle
  • on récupère tous les liens hypertextes (balise a avec attribut href)
  • on cherche le premier lien qui correspond à nos critères
  • l'url courante devient ce premier lien
  • on continue jusqu'à ce qu'on tombe sur une URL déjà explorée

On ne souhaite pas forcément explorer le tout premier lien

  • Il faut que le lien pointe sur le même hôte (ex : https://fr.wikipedia.org/gnagna) que l'url actuelle
  • Il faut que l'url ne soit pas une url interne (qui pointe vers le même fichier) de type #intra
  • Il faut que l'URL ne contienne aucun des mots de la liste passée en 2ème argument

Mémoire de l'exploration

1 point Ton explorateur doit s'arrêter dès qu'il explore une URL qu'il a déjà rencontré.

Messages pendant l'exploration

1 point Tu dois fournir des messages significatifs dans la console pour chaque exploration d'une page web contenant l'URL complète du fichier.

Sauvegarde des fichiers explorés

1 point Chaque page html explorée doit être stockée dans un fichier dans le sous-dossier resultat du dossier courant. Par exemple, le premier fichier exploré doit se trouver dans le fichier ./resultat/1.html, le deuxième dans ./resultat/2.html etc. 1 point Si le dossier resultat n'existe pas, il est créé automatiquement. Si des fichiers existent déjà dans le dossier ils sont remplacés.

1 point Chaque apparition d'une adresse courriel dans le contenu d'une page doit être remplacé par tonprenom@tonnom.com.

2 points L'attribut href de chaque page doit être modifié pour pointer de manière relative vers le prochain fichier exploré. Par exemple, dans le fichier 3.html, il faut modifier le href de la balise qui lie vers la prochaine page (4.html) pour 

<a href="4.html" ...

Collecte des courriels

1 point Tu dois collecter les adresses courriels contenues dans chaque page pendant l'exploration.

1 point L'application doit sauver la liste des courriels collectés dans un fichier appelé courriels.txt qui doit se trouver dans le dossier courant (pour un projet Maven, c'est le dossier du projet).

1 point Dans ce fichier, tu dois avoir chaque courriel sur une ligne séparé, classés dans l'ordre alphabétique. Si un courriel a été trouvé plusieurs fois dans l'exploration, tu dois supprimer les doublons avant de sauver le fichier.
