Parcourir le web

Ecrivons un programme qui parcourt des pages web en traversant à chaque fois le premier hyperlien qu'on y trouve.

L'application devra

  • Sauvegarder localement chaque fichier HTML exploré
  • Trouver les adresses courriels et les sauvegarder dans un fichier
Format de Projet Remplacer Dupont par TON nom de famille
1 point Le nom du repo GitHub privé est AXX-4203N5-ExploraWeb-Dupont (remplace Dupont par ton nom de famille, et XX par l'année). Ajoutez le professeur aux collaborateurs de ton repo.
3 points Il y a 4 commits valides dans un français correct (voir ici). Ces points sont conditionnés à la remise d'au moins un commit par semaine de cours.
0,5 point La classe principale s'appelle ExploraDupont2000.java
0,5 point Le package de l'application est dupont.explora
Correction

L'ensemble de la correction est effectuée sur le site de correction. Tu peux tester ton bot sur d'autres sites mais ça n'aura pas d'impact sur la correction.

Tu peux te servir des exemples ci-dessous pour vérifier ton code. Les exemples sont en ordre des étapes du TP.

Si tu n'es pas certain que ton code fonctionne correctement dans un exemple spécifique, demande au prof !

Arguments en ligne de commandes

Ton application doit prendre 2 arguments en ligne de commande dans l'ordre spécifié ici.

Attention! Il ne s'agit pas d'une application intéractive. Tu dois passer les arguments en ligne de commandes. Il est fortement recommandé de créer plusieurs configurations d'exécution our tester plusieurs combinaisons de paramètres corrects ou incorrects.

1 point Si l'utilisateur ne fournit pas 2 arguments, on lui indique clairement comment utiliser l'application puis l'application s'arrête.

  • quels sont les arguments
  • un exemple d'utilisation

Exemple - Argument : allo 

Merci de nous fournir 2 arguments :
l'URL de la page de départ, la liste de mots clés à ignorer séparés par des espaces entre guillemets.
Par exemple : https://info.cegepmontpetit.ca/3N5-Prog3/z/index.html "pipo popi"

1 point Si l'utilisateur a bien entré 2 arguments, il faut valider l'URL avant de commencer le traitement. Il faut afficher un message significatif si l'URL est mal formé puis quitter.

Exemple - Arguments : htps:/info.cegepmontpetit.ca/ "salut" 

Nous avons rencontré un problème avec l'URL fournie : htps:/info.cegepmontpetit.ca/
Merci de fournir une URL bien formée

1 point Si l'URL est correctement formée, il faut s'assurer qu'il existe une page à l'adresse fournie. Si ce n'est pas le cas, affichez un message puis quitter.

Exemple - Arguments : https://info.cegepmontptit.ca/ "salut" 

Il n'y a pas de page correspondante à l'URL : https://info.cegepmontptit.ca/
Merci de fournir une URL correspondant à une page.

1 point Si l'argument URL est correct, il faut s'assurer que la liste de mots clés est correcte. Nous validons qu'elle ne contient pas de répétitions.

Exemple - Arguments : https://info.cegepmontpetit.ca/ "pipo aaa pipo" 

Mots clés fournis incorrects "pipo aaa pipo".
Merci de ne pas avoir de répétitions dans les mots clés : pipo

Exploration

1 point Si les arguments sont corrects, il faut alors commencer l'exploration de la première page. Tu dois afficher dans la console le titre (title) de la page et l'URL complète.

Exemple - Arguments : https://info.cegepmontpetit.ca/3N5-Prog3/z/index.html "pipo popi" 

Les arguments sont corrects, nous commençons l'exploration de https://info.cegepmontpetit.ca/3N5-Prog3/z/index.html
Titre : Am                          URL : https://info.cegepmontpetit.ca/3N5-Prog3/z/index.html   liens 4

1 point L'exploration devrait continuer en explorant le premier lien de chaque page. L'exploration s'arrête si une page ne contient aucun lien.

Exemple - Arguments : https://info.cegepmontpetit.ca/3N5-Prog3/z/sub/aaa/index.html "allo" 

Les arguments sont corrects, nous commençons l'exploration de https://info.cegepmontpetit.ca/3N5-Prog3/z/sub/aaa/index.html
Titre : Pic et Pic                          URL : https://info.cegepmontpetit.ca/3N5-Prog3/z/sub/aaa/index.html   liens 2
Titre : La page 5                          URL : https://info.cegepmontpetit.ca/3N5-Prog3/z/sub/aaa/page5.html
L'exploration s'est arrêtée, la page https://info.cegepmontpetit.ca/3N5-Prog3/z/sub/aaa/page5.html ne contient aucun lien valide.

Sauvegarde des fichiers explorés

1 point Chaque page html explorée doit être stockée dans un fichier avec le même nom que la resource explorée (index.html, 1.html etc.).

1 point Chaque apparition d'une adresse courriel dans le contenu d'une page doit être remplacée par prenom.nom@pipo.org avant l'écriture sur le disque.

2 points Les liens relatifs entre pages téléchargées doivent être fonctionnels et doivent pointer vers les fichiers locaux si 2 pages sont dans le même dossier.

2 points Les liens relatifs doivent fonctionner entre resources locales dans des dossiers différents.

Collecte des courriels

1 point Le programme collecte les adresses courriels valides contenues dans chaque page pendant l'exploration. L'application doit sauvegarder la liste des courriels en minuscule dans un fichier dossier_du_projet/resultat/courriels.txt

1 point Dans ce fichier, tu dois avoir chaque courriel sur une ligne séparée, classés dans l'ordre alphabétique. Si un courriel a été trouvé plusieurs fois dans l'exploration, tu dois supprimer les doublons.
